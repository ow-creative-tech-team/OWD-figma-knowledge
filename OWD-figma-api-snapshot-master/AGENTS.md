# AGENTS.md

Guidance for AI agents and automation working in the `OWD-figma-api-snapshot-master/` snapshot project.

## Project Summary

This directory builds and publishes LLM-friendly snapshots of Figma developer documentation. It crawls selected sections of `https://developers.figma.com`, converts the pages to Markdown, chunks the Markdown for retrieval workflows, and checks the generated dataset into `out/`.

The project is intentionally small:

- Runtime: Node.js ES modules.
- Main entry point: `src/index.js`.
- Crawler: `src/crawler.js`.
- Shared helpers: `src/utils.js`.
- Crawl targets and tuning: `src/config.js`.
- Generated data: `out/`.
- Automation: `../.github/workflows/daily-crawl.yml`.

The checked-in dataset is part of the product, not disposable build output.

## Generated Data Contract

`out/` is generated by `npm run build`, but it is intentionally committed.

Do not manually edit generated files unless the task explicitly asks for a surgical data correction. Prefer changing crawler logic or config, then regenerating `out/`.

Generated files have these roles:

- `out/raw/<target-id>/*.json`: one JSON document per crawled source page.
- `out/llm/figma-<target-id>-corpus.md`: concatenated Markdown corpus for long-context use.
- `out/llm/figma-<target-id>-chunks.jsonl`: one JSON object per retrieval chunk.
- `out/llm/figma-<target-id>-manifest.json`: generation metadata and chunking settings.
- `out/index.json`: flat index of all crawled pages.
- `out/.complete`: local completion marker written by the build, ignored by Git.

Each JSONL chunk currently contains:

```json
{
  "id": "sha1",
  "source": "https://developers.figma.com/...",
  "title": "Page title",
  "path": "/docs/...",
  "chunkIndex": 0,
  "content": "Markdown chunk",
  "length": 1234
}
```

When changing chunk shape, slug generation, output paths, or manifest fields, update the README and consider migration impact for downstream consumers.

## Source Architecture

`src/config.js`

- Defines shared crawl defaults and `TARGETS`.
- Add new documentation sections here using:
  - `id`: output namespace, used in file names and `out/raw/<id>`.
  - `label`: human-readable dataset label.
  - `startUrl`: first page to crawl.
  - `allowedPathPrefix`: path boundary for crawler discovery.
- Keep `origin` and `allowedPathPrefix` restrictive. The crawler follows links discovered on pages, so broad prefixes can explode crawl scope.

`src/crawler.js`

- Implements breadth-first crawling with a queue and `visited` set.
- Fetches pages with `axios`.
- Extracts main content with Cheerio using selectors in `pickMainContent`.
- Removes scripts, styles, iframes, SVG, buttons, pictures, and images before Markdown conversion.
- Converts HTML to Markdown using `turndown`.
- Only captures pages whose Markdown length is greater than 80 characters.
- Discovers additional URLs from page links and filters them through `looksLikeDocUrl`.

`src/index.js`

- Processes each target in sequence.
- Writes raw JSON pages, Markdown corpus, JSONL chunks, manifest files, and `out/index.json`.
- Creates stable-ish chunk IDs from SHA-1 hashes of page URL, chunk index, and chunk content prefix.
- Deduplicates chunks by full content hash before writing JSONL.

`src/utils.js`

- Provides URL normalization, slug generation, chunking, directory creation, and file writing helpers.
- `safeSlug` maps URL paths to file-safe names by replacing path separators with `__`.
- `markdownToChunks` splits first on top-level Markdown headings, then by fixed character windows with overlap.

## Commands

Run these commands from `OWD-figma-api-snapshot-master/`.

Install dependencies:

```bash
npm ci
```

Run the crawler and regenerate `out/`:

```bash
npm run build
```

Equivalent start command:

```bash
npm start
```

Syntax-check source files without installing dependencies:

```bash
node --check src/index.js
node --check src/crawler.js
node --check src/config.js
node --check src/utils.js
```

Inspect dependency state:

```bash
npm ls --depth=0
```

There is currently no test, lint, or format script in `package.json`.

## Runtime Requirements

GitHub Actions uses Node.js 20. The current lockfile resolves `cheerio@1.2.0`, which requires Node `>=20.18.1`. Use Node 20.18.1 or newer when installing or running locally.

The crawler requires network access to `https://developers.figma.com`. In restricted or sandboxed environments, `npm ci` and `npm run build` may fail unless network access is approved.

## GitHub Actions Behavior

`../.github/workflows/daily-crawl.yml`:

- Runs daily at 03:00 UTC and on manual workflow dispatch.
- Uses `npm ci` in `OWD-figma-api-snapshot-master/`.
- Runs `npm run build` in `OWD-figma-api-snapshot-master/`.
- Commits changed `out/` files.
- Runs `npm version minor --no-git-tag-version` when data changes.
- Commits the version bump.
- Tags the new version and pushes commits/tags.
- Creates a GitHub Release for the new tag.

The workflow requires repository write permission for contents. Creating or updating files under `.github/workflows/` also requires a GitHub token with `workflow` scope when pushing from a local OAuth-authenticated client.

## Development Rules

Follow the existing style:

- ES modules only.
- Use double quotes.
- Use semicolons.
- Keep functions small and plain.
- Prefer standard Node APIs and the existing helpers in `src/utils.js`.
- Avoid adding frameworks or build tooling unless the task truly requires them.
- Keep all crawler configuration centralized in `src/config.js`.
- Keep output paths deterministic.

When changing crawler behavior:

- Preserve the current output contract unless the task explicitly asks for a breaking change.
- Keep URL normalization strict to prevent duplicate crawls.
- Avoid making the crawler more aggressive without a clear reason.
- Respect `requestDelayMs` and `maxPages`; these protect both this repo and the upstream docs site.
- Be careful when changing `pickMainContent` or cleanup rules; small extraction changes can rewrite much of `out/`.

When changing chunking:

- Keep overlap smaller than chunk size.
- Preserve deterministic ordering.
- Understand that any chunking change can rewrite all JSONL chunk IDs.
- Regenerate manifests with the new chunking settings.

When changing dependencies:

- Use `npm install` or `npm install --package-lock-only` so `package-lock.json` stays in sync.
- Do not hand-edit lockfile dependency entries.
- Keep the root `name` and `version` metadata in `package-lock.json` aligned with `package.json`.

## Search Guidance

The `out/` directory contains large generated corpora, so broad searches are noisy.

Prefer scoped searches:

```bash
rg "pattern" src README.md package.json ../.github
rg "pattern" out/index.json out/llm/*-manifest.json
```

Search `out/raw` or `out/llm/*-corpus.md` only when the task is specifically about generated content.

If dependencies are not installed or network access is unavailable, state that clearly in the final response and report any validation that did run.

## Operational Cautions

- Do not delete `out/` because it is the published dataset.
- Do not commit `node_modules/`.
- Do not commit `out/.complete`.
- Do not run the crawler repeatedly without need; it fetches hundreds of upstream pages.
- Do not broaden `allowedPathPrefix` to `/docs` or `/` unless intentionally creating a much larger dataset.
- Do not update GitHub workflow files unless the user can push with `workflow` scope.
- Do not mix generated dataset refreshes with unrelated source refactors in the same commit unless explicitly requested.

## Common Tasks

Add a new docs section:

1. Add a target to `TARGETS` in `src/config.js`.
2. Ensure `id` is stable and file-name safe.
3. Run `npm run build`.
4. Confirm new `out/raw/<id>`, `out/llm/figma-<id>-*`, and `out/index.json` entries.
5. Update README output tables if the new dataset should be advertised.

Tune chunk size:

1. Update `chunkSizeChars` and/or `chunkOverlapChars` in `src/config.js`.
2. Run `npm run build`.
3. Expect JSONL files and manifests to change broadly.
4. Check downstream compatibility because chunk IDs may change.

Improve extraction quality:

1. Adjust `pickMainContent`, `cleanupNode`, or `htmlToMarkdown` in `src/crawler.js`.
2. Run a full build.
3. Compare `git diff --stat` and sample raw/corpus output before committing.

Refresh dataset only:

1. Run `npm ci` if dependencies are missing.
2. Run `npm run build`.
3. Commit only `out/` changes unless versioning or workflow requirements say otherwise.
